재고 관리 프로그램

목표 : 프로그램 형식으로 [부품 재고현황]파일 재구현.

0. UI나 상세 설계 등은 자율적으로 구상해도 무방. 납품명세서는 항상 동일한 형식으로 제공된다고 가정.
1. [납품명세서 상세]를 프로그램에 업로드
2. 품번을 기준으로, 재고현황 업데이트(재고-납품량)
3. 업데이트 완료되면 오늘 날짜([부품 재고현황] G1셀)로 데이터 저장
4. 재고현황 대시보드 제작(차트 형식으로, 또는 주간, 월간, 분기, 반기, 년간으로 데이터 집계만 해줘도 무방. 자율적으로 대시보드 제작)

* 추후 재고가 생산될 경우에도 추가해야함.
* 관리자가 수동으로 재고를 수정할 수 있도록 확장성 고려(수동 수정은 수동모드를 켜야하거나 수정시 경고 필요).

대시보드 HTML
HTML에 납품명세서 엑셀파일을 끌어오면 업로드 되어서 자동으로 재고현황을 업데이트 하는 로직
재고현황 엑셀파일이 DB역할
일부 기업에서는 파이썬 라이브러리를 막아두는 경우도 있음

Power automate 연동?

++ 가능하다면 기업환경에서 화면 인식이 된다면 명세서를 바로 인식하도록

납품 명세서 파일을 업로드 받으면 해당 파일에 날짜를 붙여서 별도로 저장
이 CSV 파일들을 토대로 대시보드 자료 완성
어떤 데이터를 어떤 그래프로 보여줄건지, 여러 데이터들로 그래프 여러개 출력

대시보드가 메인, 세부 데이터 보기 및 데이터 입력이 서브
